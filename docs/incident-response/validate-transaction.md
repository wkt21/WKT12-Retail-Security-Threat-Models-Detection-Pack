# Step 2 — Validate the Transaction

## Purpose
Confirm whether the transaction data aligns with expected pricing, discount logic, and SKU metadata.

## Validation Checklist
- Expected vs actual price
- Expected vs actual discount
- SKU ↔ department match
- Coupon category and expiration
- Scan cadence and timing
- POS workflow integrity

## SOC Actions
- Compare transaction logs to product database.
- Review coupon parser output.
- Check for discount logic anomalies.
- Verify SKU metadata against department codes.
